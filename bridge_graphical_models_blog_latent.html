Bridge Graphical Models

From learning latent variables to designing latent paths for diffusion, flow matching, Schrödinger bridges, and field-based generative models.
Author: Tiantian ZHANG Status: under review at NeurIPS
A bridge is not yet a generator: a latent path must be compressible into a Markov decoder.

0. Start from latent learning

A simple way to read this project is: generative modeling is about choosing latent variables that make data generation easier. A VAE learns a latent code; a diffusion model fixes a noisy latent path; Bridge Graphical Models ask how to choose the whole latent path so that it can be decoded by a Markov sampler.

VAE. The encoder \(q_\phi(z\mid x)\) chooses a latent code \(z\) that should both explain the data and remain close to a simple prior. The design question is whether \(z\) contains the right information about \(x\).

Diffusion. The latent variable becomes a trajectory \(X_{0:1}\), usually produced by a fixed Gaussian noising process. The model learns to denoise or predict velocity along this prescribed noisy path.

BGM. The endpoint coupling and bridge law are themselves latent-path design choices. A good latent path should not only connect the base and data distributions; it should also retain enough information for a non-anticipative Markov decoder to know how to move.

\[ \text{latent code learning} \quad\longrightarrow\quad \text{latent path design} \quad\longrightarrow\quad \text{Markov-decodable latent paths.} \]

The Markovization gap measures the last requirement. It asks whether the local bridge velocity \(U_t\) can be predicted from the sampler's state \(S_t=(X_t,t)\). If not, the missing endpoint information becomes an irreducible training target error before architecture, optimizer, or solver choices enter.

1. The core bottleneck

Continuous-time generative models often begin by defining paths between a simple base distribution and the data distribution. Diffusion models use noisy paths, flow matching uses conditional probability paths, rectified flow often uses straight-line paths, and Poisson or electrostatic methods use field lines in augmented space. These choices look different, but they share a structural constraint: the path used for training may know both endpoints, while the sampler used at generation time cannot.

At training time, a bridge can depend on a base endpoint \(Z\) and a data endpoint \(X\). At generation time, a Markov decoder only sees the current state and time, \(S_t=(X_t,t)\). If many endpoint pairs pass through similar intermediate states with incompatible velocities, the missing endpoint information cannot be recovered by increasing neural-network capacity.

The paper's central claim is that bridge-to-Markov compression is a structural bottleneck, and that this bottleneck can be measured before training.

2. From VAE to diffusion to Bridge Graphical Models

The VAE viewpoint is useful because it separates two tasks: infer latent variables and decode data from them. A standard VAE learns \(q_\phi(z\mid x)\) and trains the decoder so that \(z\) is both informative and prior-compatible. Diffusion models can be read as a path-space version of this idea: the latent variable is no longer one code, but a sequence of noisy states. In most diffusion models, this path is fixed by a Gaussian corruption schedule.

BGMs generalize this by treating the latent path itself as the object to design. The endpoint coupling decides which base samples and data samples are paired. The bridge law decides how information moves between the paired endpoints. The Markov decoder is what remains after this endpoint-conditioned path is compressed into dynamics that only observe the current state and time.

\[ \text{VAE: } q_\phi(z\mid x) \quad\longrightarrow\quad \text{Diffusion: } q(X_{0:1}\mid x) \quad\longrightarrow\quad \text{BGM: } \scrQ_{\Gamma,\lambda}(X_{0:1}). \]

This adds a criterion that is absent from the usual latent-code story. The latent path should be Markov-decodable:

\[ U_t \approx \mathbb E[U_t\mid X_t,t]. \]

If this approximation is poor, the bridge may still be mathematically valid, but it is a bad training target for a Markov sampler. The Markovization gap quantifies exactly this failure mode.

3. The BGM factorization

A Bridge Graphical Model separates four choices that are usually entangled:

Endpoint coupling \(\Gamma\)

How base points and data points are paired: independent coupling, minibatch OT, entropic OT, or a learned encoder coupling.

Bridge law \(\mathsf R^{z,x}\)

The conditional path distribution between endpoints: straight, Gaussian, Brownian, Schrödinger, or field-line bridge.

Markovian projection

The operation that turns endpoint-conditioned local motion into a non-anticipative velocity field.

Dynamics representation

The way the current is sampled: ODE, SDE with score correction, or augmented field dynamics.

\[ \mathsf Q_{\Gamma,\lambda}(dX_{0:1})= \int \mathsf R^{z,x}_\lambda(dX_{0:1})\,\Gamma(dz,dx). \]

The factorization makes it possible to ask which design choice creates the learning burden. The Markovization gap targets the projection box.

4. Markovian projection and the Markovization gap

Let \(U_t\) denote the endpoint-conditioned bridge velocity. A Markov decoder uses a vector field \(v(X_t,t)\), so the natural population loss is

\[ \mathcal L_{\mathrm{FM}}(v)=\int_0^1 \mathbb E\|U_t-v(X_t,t)\|^2\,dt. \]

By the Hilbert-space projection theorem, the optimal Markov velocity is the conditional expectation

\[ v^\star(x,t)=\mathbb E[U_t\mid X_t=x]. \]

The loss decomposes as

\[ \mathcal L_{\mathrm{FM}}(v)= \underbrace{\int_0^1\mathbb E\operatorname{Tr}\operatorname{Var}(U_t\mid X_t)\,dt}_{\mathfrak G(\mathsf Q)} + \int_0^1\mathbb E\|v^\star(X_t,t)-v(X_t,t)\|^2\,dt. \]

The first term is the Markovization gap. It is the regression floor imposed by the chosen coupling and bridge law before any finite architecture is selected.

5. Information-theoretic interpretation

The Markovization gap is an information-bottleneck quantity in a task-specific squared-loss sense. Let \(H=(Z,X)\) be hidden endpoint information and \(S_t=(X_t,t)\) the state observed by the sampler. The bridge velocity can depend on \(H\), but the Markov sampler can only use \(S_t\). Therefore

\[ \mathfrak G(\mathsf Q)=\int_0^1\mathbb E\|U_t-\mathbb E[U_t\mid S_t]\|^2\,dt. \]

This is the MMSE of predicting endpoint-conditioned motion from the information available to the sampler. It does not claim to be Shannon mutual information. It measures the velocity-relevant endpoint information that does not survive the Markov bottleneck.

A normalized version can control for raw velocity scale:

\[ \eta(\mathsf Q)=\frac{\int_0^1\mathbb E\operatorname{Tr}\operatorname{Var}(U_t\mid S_t)\,dt}{\int_0^1\mathbb E\operatorname{Tr}\operatorname{Var}(U_t)\,dt}. \]

6. Rectified flow and the “not learning the optimal” issue

For a straight bridge,

\[ X_t=(1-t)Z+tX, \qquad U_t=X-Z. \]

Rectified flow and flow matching do not learn the endpoint displacement \(X-Z\) directly at sampling time. They learn the best Markov projection for the chosen path:

\[ v^\star(x,t)=\mathbb E[X-Z\mid X_t=x]. \]

If the coupling creates many crossings, this conditional mean is still the optimal Markov field, but it is forced to average incompatible endpoint velocities. The failure is not that regression misses its optimum. The deeper issue is that the chosen latent path is not Markov-decodable.

BGM resolves this by shifting the design problem upstream: choose or learn a coupling, bridge, or augmented state representation that reduces \(\mathfrak G(\mathsf Q)\) before training.

7. Current-preserving dynamics representation

The probability path and the sampler representation are separate. If \(\rho_t\) satisfies

\[ \partial_t\rho_t+\nabla\cdot(\rho_t v_t)=0, \qquad J_t=\rho_t v_t, \]

then an SDE with diffusion covariance \(a(x,t)\) can realize the same marginal path by choosing

\[ b_i^{(a)}(x,t)=v_i(x,t)+\frac{1}{2\rho_t(x)}\sum_j\partial_j(a_{ij}(x,t)\rho_t(x)). \]

For isotropic diffusion \(a=g(t)^2I\), this becomes a score correction. This identity is standard Fokker--Planck algebra, but in BGM it plays an important organizational role: the bridge defines a current, while the sampler chooses how to represent it.

8. Gaussian paths: score, noise, and velocity

For Gaussian interpolation

\[ X_t=\alpha_t X+\sigma_t\epsilon,\qquad \epsilon\sim\mathcal N(0,I), \]

Tweedie's identity gives

\[ \mathbb E[\epsilon\mid X_t=x] = -\sigma_t\nabla\log\rho_t(x). \]

The projected velocity is

\[ v^\star(x,t)=\frac{\dot\alpha_t}{\alpha_t}x+ \left(\frac{\dot\alpha_t\sigma_t^2}{\alpha_t}-\dot\sigma_t\sigma_t\right)\nabla\log\rho_t(x). \]

Thus score, noise, denoiser, and velocity prediction are schedule-dependent reparameterizations of the same conditional statistic under a chosen Gaussian path. BGM asks whether that path itself is a good Markov-decodable latent path.

9. Field-line and Poisson bridges

Field-based generative models can be represented as bridge kernels. Let \(\Phi\) be a potential, \(F=-\nabla\Phi\), and \(\varphi_s\) solve \(\dot\varphi_s=F(\varphi_s)\). If \(T_\Phi(z)\) is the hitting point on the data surface, then

\[ \Gamma_\Phi=(\operatorname{Id},T_\Phi)_\#\mu_0, \qquad \mathsf R_\Phi^{z,x}=\delta_{\gamma_z}. \]

This gives field methods their own Markovization gap. If multiple field lines collapse to the same observed state with different velocities, the data-space Markov decoder loses information. An augmented field representation can retain it.

10. Empirical story

The experiments test one controlled claim: a cheap proxy Markovization gap should rank bridge or coupling choices that are easier to compress into a Markov decoder.

11. What BGM adds over existing unifications

Flow Matching formulates simulation-free training by regressing vector fields of conditional probability paths. Generator Matching broadens the Markov-process language by learning infinitesimal generators. Stochastic interpolants unify flows and diffusions through interpolating processes and currents. These frameworks describe how to construct or learn a Markov process.

BGM asks a complementary question: once a bridge or conditional generator target is specified, how much endpoint-conditioned information is lost when it is compressed into a Markov decoder?

12. Practical design rules

The framework suggests four practical rules.

Do not choose paths only by geometry

A straight path can be short but hard to Markovize if endpoint-conditioned paths cross.

Treat coupling as inference

Independent coupling leaves endpoint alignment to the decoder. OT or learned encoders can move some alignment into the latent path.

Separate path from representation

The same probability current can be sampled as an ODE, SDE, or augmented field.

Screen before training

Estimate a proxy gap before committing GPU time to a bridge/coupling design.

A good latent path is one whose velocity is nearly measurable from the Markov state.
